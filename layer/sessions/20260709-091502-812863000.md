**Discussion context:**
- The user observed that the project was accumulating too much tech debt through parallel tutor modes and wanted to move on to the new method.
- We reframed the product goal from “guided lesson UI” to “Rustlings with training”: a learner should progress through tiny failing specs, deterministic checks, hints, Pi explanations, and eventually capstone/review gates.
- The reasoning framework was to keep Pi as a helper/coach, not the primary UI, and make deterministic Allium exercise completion the durable proof of progress.

**Key decisions:**
- Canonical command is now `./learn-allium`, which means Alliumlings by default; `./learn-allium watch` is retained as an alias rather than a separate conceptual mode.
- Legacy modes remain accessible but are deliberately named as legacy/fallback so learners do not have to choose among multiple “correct” paths.
- The proficiency path should avoid overclaiming certification. Automated completion can support “Allium-proficient”; “expert” should require capstone artifacts and review.
- The Alliumlings path continues the [[local-examples-first]] belief: exercises are local, copied into isolated run workspaces, and not dependent on external production repos.

**Challenges faced:**
- The earlier architecture had overlapping modes: Pi extension tutor, standalone tmux tutor, and Alliumlings. The fix was not more features but a simpler routing rule: default to Alliumlings, keep older work as fallback.
- `scripts/allium-tutor` needed careful alias changes so smoke tests still worked: `./learn-allium --smoke` now exercises Alliumlings, and `./learn-allium lesson-01 --smoke` still validates the old single lesson.
- Documentation needed terminology cleanup so classroom instructions no longer imply Lesson 01 is the main experience.

**Patterns observed:**
- For training tools, the default command should encode the canonical pedagogy; optional modes should not compete with the primary path.
- A Rustlings-style exercise runner lowers learner cognitive load better than a large guided menu: default action is “check my work,” with hints and Pi help nearby.
- The agent belongs beside the exercise loop as a coach, while deterministic `allium check`/`allium analyse` remain the gate.
